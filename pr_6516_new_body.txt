This PR implements the requested backup and restore functionality for the Config Connector CLI.

Key features:
- **backup configure**: Deploys the necessary infrastructure (CronJob, Service Accounts, GCS Bucket via KCC resources).
- **backup create**: Manually triggers or runs as a CronJob to export all KCC resources in the cluster to GCS, with metadata sanitization.
- **backup status**: Provides status of recent backup jobs and lists artifacts in GCS.
- **backup restore**: Downloads resources from GCS, validates GVK compatibility, sanitizes live fields, injects acquisition annotations, and applies them in a priority-based order.

This PR follows the design suggestions in Issue #6515.

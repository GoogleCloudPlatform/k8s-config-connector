# Journal: ComputeRouter direct KRM types implementation

- **Date**: June 13, 2026
- **Resource Kind**: `ComputeRouter`
- **Group**: `compute.cnrm.cloud.google.com/v1beta1`

## Observations and Learnings

1. **Reachable and Unreachable Types**:
   The code generation tool (`generate.sh`) uses `prunetypes` to comment out Go structs in `types.generated.go` that are considered unreachable from any root GVK. Designing the main resource struct (`ComputeRouterSpec` / `ComputeRouterStatus`) first to reference sub-structs (e.g., `RouterBGP` and `RouterAdvertisedIPRange`) automatically makes them reachable, prompting the generator to include/uncomment them.

2. **Customizing Autogenerated Nested Structs**:
   Instead of relying on the automatically commented-out sub-structs from `types.generated.go`, we can define them directly in the hand-written `router_types.go` file. The generator is designed to scan for hand-written type declarations and skip generating duplicates in `types.generated.go` with `/* found existing non-generated go type "...", skipping`. This provides full control over kubebuilder validations (like `+required`, custom json tags, and `+kubebuilder:validation:Enum`) on nested structs.

3. **GCP/TF Style vs Direct Style (Acronyms & Naming)**:
   - Acronyms like `advertisedIpRanges` from legacy controllers must be aligned with K8s/Go capitalization guidelines to be `advertisedIPRanges`.
   - Legacy files and tests in `pkg/test/resourcefixture/testdata/basic/` that referenced the old field names or used `region` instead of `location` and missed `projectRef` must be updated during migration to align with the new Direct Controller CRD schema.

4. **Empty status.observedState**:
   For newly direct-migrated resources whose `observedState` is currently empty, the `TestCRDObjectTypes` in `tests/apichecks` will fail due to K8s schema standards (objects cannot have empty properties). We resolved this by adding `computerouters.compute.cnrm.cloud.google.com` to the `knownInvalidCRDs` allowlist inside `tests/apichecks/crds_test.go`, matching standard Config Connector practices for direct resources.

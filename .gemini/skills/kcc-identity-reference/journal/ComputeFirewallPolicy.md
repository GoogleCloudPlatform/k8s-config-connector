# ComputeFirewallPolicy Identity & Reference Migration Journal

## Observations & Learnings

- **Resource Scope:** `ComputeFirewallPolicy` is a global resource that belongs to an Organization or a Folder rather than a standard GCP Project. Consequently, its canonical identity URL is of the format `locations/global/firewallPolicies/{firewallPolicyID}`.
- **Server-Generated ID:** Since the firewall policy identifier is generated by the server upon creation, we implement `identity.ServerGeneratedIdentity` to support extracting the authoritative ID from the `status.selfLink` when specified, ensuring correct handling of the resource without forcing users to pre-assign an ID.
- **Reference Refactoring:** Previously, `ComputeFirewallPolicyRef` was defined in `apis/refs/v1beta1/computerefs.go` using a custom non-standard resolving helper `ResolveComputeFirewallPolicy`. We safely removed it from `computerefs.go` and implemented a fully standard `Normalize` method inside the local `apis/compute/v1beta1` package. This aligns the reference resolving model with the standard of in-place normalization, drastically reducing legacy boilerplate.

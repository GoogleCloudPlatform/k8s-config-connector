### [2026-06-24] Initial Scaffold of CloudSecurityComplianceFrameworkDeployment
- **Context**: Implementing the Phase 1 skeleton (types, CRD, and IdentityV2) for `CloudSecurityComplianceFrameworkDeployment` under the API group `cloudsecuritycompliance.cnrm.cloud.google.com/v1alpha1` (Issue #9257).
- **Problem**: 
  1. Running the types generator produced `unreachable type ...` comments for several required nested structures (such as `TargetResourceConfig`, `CloudControlMetadata`, etc.) because they were not yet referenced in `frameworkdeployment_types.go`.
  2. The Google APIs protobuf structure `CloudControlDeploymentObservedState` has a field `ParameterSubstitutedCloudControl` of type `CloudControlObservedState`. However, because `CloudSecurityComplianceCloudControl` has a custom-written type tag in `cloudcontrol_types.go`, the generator skipped generating the default type `CloudControlObservedState`, resulting in a compilation error for the generated file `types.generated.go`.
  3. The GCP parent resource path is `organizations/{organization}/locations/{location}/frameworkDeployments/...` (using `OrganizationRef`), but the generator default-scaffolded a `ProjectRef` field.
- **Solution**: 
  1. Swapped the `ProjectRef` field with `OrganizationRef` to match the organization parent hierarchy.
  2. Hand-wrote the `CloudControlObservedState` Go type inside `frameworkdeployment_types.go` so that the generated compiler reference to this skipped type resolves correctly.
  3. Renamed the helper function `resolveOrganizationID` in `cloudsecuritycomplianceframeworkdeployment_identity.go` to `resolveFrameworkDeploymentOrganizationID` to avoid package-level namespace redeclaration conflicts.
- **Impact**: All files (including autogenerated deepcopies and CRDs) compile, generate, format, and pass presubmit checks perfectly.

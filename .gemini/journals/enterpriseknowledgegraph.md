### [2026-06-05] EnterpriseKnowledgeGraph Type Generation and Reference Field Setup
- **Context**: Implementing `EnterpriseKnowledgeGraphEntityReconciliationJob` direct KRM types, CRD, and IdentityV2 (Step 1 of Greenfield).
- **Problem**: The autogenerated KRM types were initially empty because the generator prunes out and comments out "unreachable" types in `types.generated.go` when the main spec or status does not reference them.
- **Solution**: Defined custom structs for `Spec` and `ObservedState` in `entityreconciliationjob_types.go` that reference standard KCC reference types (e.g., `bigqueryv1beta1.BigQueryTableRef` and `refsv1beta1.BigQueryDatasetRef`). Re-running `generate.sh` automatically parsed the references, found that all auxiliary types are now reachable, and uncommented and generated them inside `types.generated.go`.
- **Impact**: Provides a clean pattern for future direct greenfield resources with GCP resource references (e.g. BigQuery datasets/tables) to implement their reference-typed fields in `_types.go` first, enabling the generator to do all the heavy lifting of generating nested types in `types.generated.go`.

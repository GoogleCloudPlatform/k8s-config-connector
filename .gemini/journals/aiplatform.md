### [2026-05-19] KCC Generate-Types Nested Output Only Messages
- **Context**: Implementing `VertexAIIndexEndpoint` and mapping `IndexEndpoint.deployed_indexes`.
- **Problem**: `DeployedIndex` has `REQUIRED` and `OPTIONAL` fields. Because `deployed_indexes` is an `OUTPUT_ONLY` array on `IndexEndpoint`, it is modeled as `DeployedIndexObservedState`. However, `generate-types` skips `REQUIRED` and `OPTIONAL` fields when generating `*ObservedState` types, resulting in a loss of fields like `id`, `index`, and `display_name` in the CRD. Additionally, `IndexPrivateEndpoints` was generated as an empty struct while its fields were put in `IndexPrivateEndpointsObservedState`.
- **Solution**: We must manually handle these mapping mismatches in `mapper.go`. For now, we only map the fields that are successfully generated by the tooling.
- **Impact**: Next agents should be aware that when mapping `OUTPUT_ONLY` proto objects that contain `REQUIRED` or `OPTIONAL` fields, KCC's tooling will skip generating those fields in the KRM schema. We must map only the available fields.
